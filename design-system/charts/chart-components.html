Chart Designs

Trend-focused weight charts designed for every scenario — different weighing habits, goal types, and data patterns. The trend line is always the primary visual; raw data points are secondary.

Chart Anatomy

Every chart shares the same visual hierarchy: bold trend line on top, subtle raw data underneath. Goal and status information is encoded through colour, not clutter.

Visual Hierarchy
How the layers compose together
● On track

Raw data points
Daily range (multi-weigh)

Layer order (back→front): Goal band fill → Grid → Daily range bars → Raw dots → Trend fill → Trend line
Trend line: 3px solid where data exists, 2px dashed at 35% opacity through gaps
Raw dots: 3px radius, tertiary colour at 40% opacity — visible but never competing
Range bars: 5px wide, trend colour at 18% opacity — shows daily min→max for multi-weigh days
Goal line: 1.5px dashed, info colour — steady reference that doesn't dominate
Gaps: Trend line switches from solid to dashed — no additional decoration needed

Chart Design Rules

Codified rules for consistent rendering across all scenarios.

TREND ALGORITHM Exponentially Weighted Moving Average (EWMA), α=0.15 → Smooths out daily ±0.5–1.0 kg water fluctuations → Responds to genuine change within ~7 days COLOUR LOGIC (when goal is set) Green (on-track): Trend moving towards goal, or within maintenance band Amber (unclear): Trend flat when loss/gain expected, or within 20% of maintenance boundary Red (off-track): Trend moving away from goal, or outside maintenance band COLOUR TRANSITION Trend line can change colour along its length via gradient stops → Reflects status changes mid-period (e.g. on-track → stalling) → Transition occurs at the point where status changes DATA GAP HANDLING Trend carries forward through gaps (your weight doesn't vanish) Gap segments drawn dashed at 35% opacity → clearly "estimated, not measured" When a new measurement arrives, EWMA resumes from the carried value SPARSE DATA (<5 entries) Entire trend line drawn dashed at 45% opacity → "low confidence" Badge shows "Not enough data" rather than a directional status Encourages more frequent weighing without hiding what data exists MULTI-WEIGH DAYS Show vertical range bar (min→max) at 18% opacity Plot average as the primary data point Trend calculations use the daily average Y-AXIS SCALING Auto-scale to data range + 10% padding top/bottom If goal exists, always include goal line in visible range Maintain minimum 2kg visible range (prevents noisy appearance) X-AXIS Show date labels at regular intervals (daily, weekly, or monthly depending on range) Weekend shading optional (subtle bg tint) Today marker: vertical dotted line ACCESSIBILITY All colours meet 3:1 minimum on chart background (non-text contrast) Trend status conveyed by badge text + icon, never colour alone Screen reader: provide sr-only table of data points as alternative Chart role="img" with descriptive aria-label
